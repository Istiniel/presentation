JSON web tockens (JWT)

The concept of JSON Web Tokens and how to impement it

JWT structure

JWT consists of three parts

Header

Payload

Signature

Header

Service part, JWT "settings" (algorithm, token type)


					{
						"alg": "HS256",
						"typ": "JWT"
					}

Payload

Identifying information (user id, name etc)

here can be stored any useful data

					
					{
						"sub": "1234567890",
						"name": "Quincy Larson",
						"iat": 1516239022
					}

"iat" is the "issued at" date for the token, and is a registered claim

Signature

Cryptographic signature of the token

The header and payload parts are encrypted by base64url algorithm and afterwards united in a single box using a dot (“.”) as a divider 

					
						HMACSHA256(
							base64UrlEncode(header) + "." +
							base64UrlEncode(payload),
							secret-key
						)

Secret key is used to generate JWT signature. It has to be stored serverside and shared beetwen backend services
